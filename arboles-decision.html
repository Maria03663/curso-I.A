Árboles de Decisión y Random Forest

Dominá los algoritmos de árboles: desde un simple árbol de decisión hasta poderosos ensambles como Random Forest y Gradient Boosting.

Árboles de Decisión: Teoría
Teoría Nodos, entropía, Gini, ganancia de información

Un árbol de decisión es un modelo predictivo que toma decisiones basándose en una serie de preguntas sobre los atributos de los datos. Cada nodo interno representa una condición sobre un atributo, cada rama representa el resultado de esa condición, y cada hoja representa una predicción final (clasificación o regresión).

Para decidir qué atributo usar en cada split, se utilizan métricas de impureza. La entropía mide el desorden de los datos: H(S) = -Σ pᵢ log₂(pᵢ). La impureza de Gini mide la probabilidad de clasificar mal un elemento si se etiquetara aleatoriamente según la distribución de clases: Gini = 1 - Σ pᵢ². La ganancia de información mide cuánto reduce la entropía un split particular.

Ejemplo numérico: Dataset con 14 instancias (9 "Sí", 5 "No"). Entropía inicial = 0.94. Al hacer split por "Tiempo" (soleado/nublado/lluvia), la entropía baja a 0.69. Ganancia = 0.94 - 0.69 = 0.25 bits. Repetimos para cada atributo y elegimos el de mayor ganancia.
Duración estimada: 55 min
2
Entrenamiento y Poda
Algoritmo ID3, C4.5, CART, overfitting, poda pre y post

Existen varios algoritmos clásicos para construir árboles. ID3 (Iterative Dichotomiser 3) usa entropía y ganancia de información, pero solo maneja atributos categóricos y no tiene poda. C4.5 es su evolución: soporta atributos continuos, valores faltantes, y usa gain ratio para evitar el sesgo hacia atributos con muchos valores. CART (Classification and Regression Trees) usa Gini para clasificación y MSE para regresión, y construye siempre árboles binarios.

Los árboles tienden al overfitting: si dejamos que crezcan sin control, memorizan los datos de entrenamiento y no generalizan bien. Los criterios de parada (max_depth, min_samples_split, min_samples_leaf) detienen el crecimiento temprano. La poda reduce el tamaño de un árbol ya crecido: pre-poda (parar antes) y post-poda (recortar después con reduced error pruning o cost complexity pruning que agrega un penalty α por cada hoja extra).

Cost Complexity Pruning: Busca minimizar Error(T) + α · |hojas|. Con α = 0 obtenemos el árbol completo (overfitting). Al aumentar α penalizamos la complejidad, obteniendo árboles más chicos y generalizables. Se elige el α óptimo por validación cruzada.
Duración estimada: 50 min
3
Random Forest y Bagging
Ensemble Bootstrap, feature sampling, OOB error, importancia

El ensemble learning combina múltiples modelos para obtener predicciones más robustas que cualquier modelo individual. La idea clave: muchos modelos débiles (weak learners) que cometen errores distintos, al combinarse, producen un predictor fuerte. El bootstrap aggregating (bagging) entrena cada modelo con una muestra distinta de los datos, obtenida por muestreo con reemplazo del dataset original.

Random Forest extiende el bagging agregando aleatoriedad en los atributos: en cada split de cada árbol, solo se considera un subconjunto aleatorio de features (típicamente √p para clasificación, p/3 para regresión). Esto decorrelaciona los árboles, reduciendo la varianza del ensemble aún más. Además, Random Forest proporciona feature importance midiendo cuánto disminuye la impureza cada atributo en promedio, y usa el Out-of-Bag (OOB) error como estimación interna del error de generalización sin necesidad de validación cruzada.

Ventajas sobre un solo árbol: Mayor precisión, menor overfitting, robustez ante outliers y ruido, estimación interna del error (OOB), ranking de importancia de variables. El costo es la pérdida de interpretabilidad directa: ya no podés dibujar un solo árbol para explicar la decisión.
Duración estimada: 60 min
4
Gradient Boosting y XGBoost
Avanzado AdaBoost, GBM, XGBoost, LightGBM, comparación

A diferencia del bagging (árboles independientes en paralelo), el boosting entrena árboles secuencialmente: cada nuevo árbol intenta corregir los errores del anterior. AdaBoost (Adaptive Boosting) ajusta los pesos de las instancias mal clasificadas para que el siguiente modelo les preste más atención. Gradient Boosting Machines (GBM) generaliza esta idea: cada nuevo árbol se entrena sobre los residuos (errores) del ensemble actual, haciendo un descenso por gradiente funcional.

XGBoost (eXtreme Gradient Boosting) lleva el GBM al extremo: añade regularización L1/L2 para evitar overfitting, paralelización a nivel de features, manejo eficiente de valores faltantes, y pruning con ganancia negativa. LightGBM usa técnicas de histograma y crecimiento leaf-wise (en lugar de level-wise) para ser aún más rápido en datasets grandes. Ambas librerías dominan las competencias de Kaggle.

Random Forest vs Boosting: Random Forest es más robusto con pocos datos y menos hiperparámetros que tunear. Boosting (XGBoost/LightGBM) suele dar mejor accuracy si tenés suficientes datos y tiempo para tunear. RF entrena en paralelo (rápido con muchos cores); boosting es secuencial (más lento). Para datos ruidosos, RF es más estable; para datos limpios con señal clara, boosting gana.
Duración estimada: 65 min
